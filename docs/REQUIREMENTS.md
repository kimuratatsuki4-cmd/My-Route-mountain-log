# システム開発参照資料：My Route & Mountain Log

## 1. アプリ概要

  * **サービス名**: My Route & Mountain Log
  * **概要**: 登山愛好家向けの登山記録・山情報管理Webプラットフォーム

## 2. 案件の背景

近年、登山やハイキングを趣味とする人口が増加傾向にある。しかし、自分自身の登山記録（登った山、歩行距離、累積標高など）を体系的に管理し、山の位置情報をマップ上で可視化できるサービスは限られている。また、登山前の天気確認や持ち物リスト（ギアリスト）の共有といった実用的な機能が統合されたプラットフォームは少ない。そこで、登山記録の蓄積・可視化から、山の天気予報やギアガイドまでをワンストップで提供するWebアプリケーションを開発するに至った。

## 3. 前提情報

  * アプリ上で登山活動の記録（Activity Log）を作成・管理できるが、外部の登山届や予約サービスとの連携は行わない。
  * 月額制の有料（Premium）会員は全機能を使えるが、無料（General）会員は一部機能しか使えない。
  * 有料会員になれば、活動統計の閲覧、おすすめの山の表示、リアルタイム天気予報の確認、ギアガイドの閲覧、準備メモの登録ができるようになる。
  * 有料会員の決済にはStripe（クレジットカード）を利用する。
  * 管理者は管理ダッシュボードで集計情報の閲覧、ユーザー管理、山データ（マスタ）管理を行うことができる。
  * 多言語対応（日本語・英語）をサポートしている。

## 4. 用語の定義

  * **非会員**: 本アプリに登録していないユーザー
  * **会員**: 本アプリにアカウント登録しているユーザー
  * **無料会員（ROLE_GENERAL）**: 会員の中でも一部機能が制限されているユーザー
  * **有料会員（ROLE_PREMIUM）**: サブスクリプション契約を結び、全機能を利用できるユーザー
  * **管理者（ROLE_ADMIN）**: 本アプリのシステム管理を行うユーザー
  * **山（Mountain）**: 山マスタデータとして登録されている山の情報（名前、標高、緯度経度、難易度など）
  * **活動ログ（Activity）**: ユーザーが記録した登山活動の基本情報（登山日、タイトル、メモ、写真等）
  * **活動詳細（ActivityDetail）**: 活動ログに紐づく詳細な数値データ（歩行距離、活動時間、累積標高、最高地点等）
  * **準備メモ（MountainNote）**: 特定の山に対してユーザーが投稿するギアリスト・持ち物メモ
  * **メモアイテム（NoteItem）**: 準備メモ内の個別の持ち物アイテム
  * **経験レベル（ExperienceLevel）**: ユーザーの登山経験レベル（BEGINNER / INTERMEDIATE / ADVANCED）
  * **決済**: Stripe ASPを利用し、クレジットカードでサブスクリプション課金を行うこと

## 5. 会員側実装機能

### 山情報

  * 山名（キーワード）による山の検索機能
  * 都道府県、難易度、標高帯による山の絞り込み機能
  * 山の一覧表示（ページネーション対応）
  * 山の詳細ページにおける基本情報（標高、県名、難易度、百名山フラグ等）の表示
  * 山の詳細ページにおける自分の過去の活動ログの表示
  * 山の詳細ページにおける「みんなの準備メモ」の表示
  * マップAPI（Leaflet.js）による山の位置情報の地図表示機能

### 活動ログ（Activity Log）

  * 活動ログの一覧表示（ソート機能付き：日付順）
  * 活動ログの新規登録機能（タイトル、日付、メモ、写真アップロード、山マスタ紐付け）
  * 活動ログの詳細表示機能
  * 活動ログの編集機能
  * 活動ログの削除機能
  * 活動ログへの活動詳細（距離・時間・累積標高・最高地点）の登録

### 会員登録・認証

  * メール認証機能（トークンによるメールアドレスの有効性確認）
  * メールアドレスとパスワードによるログイン機能
  * ユーザー名・メールアドレスの重複チェック
  * パスワード確認入力の一致チェック

### 会員情報管理

  * 会員情報の表示（マイページ）
  * 会員情報の編集機能（ユーザー名、メールアドレス、住所、生年月日、経験レベル）

### 有料会員（Premium）限定機能

  * サブスクリプション契約機能（Stripe Checkoutによる決済）
  * サブスクリプション解約機能
  * クレジットカード情報の変更機能（Stripe Elements連携）
  * 活動統計（Activity Stats）のダッシュボード表示
  * おすすめの山の表示機能
  * 過去のランダムな活動記録の表示機能
  * リアルタイム天気予報の確認機能（OpenWeatherMap API連携）
  * ギアガイド（Gear Guide）ページの閲覧
  * 準備メモ（MountainNote）の投稿機能（ギアリストの共有）

## 6. 管理者側実装機能

### 認証機能

  * メールアドレスとパスワードによるログイン機能（ROLE_ADMIN権限チェック）

### 管理ダッシュボード

  * 総会員数の集計・表示
  * 有料会員数の集計・表示
  * 登録山データ数の集計・表示
  * 直近の活動ログ数の集計・表示
  * 月間売上予測データの表示

### ユーザー管理

  * ユーザーの一覧表示（ページネーション対応）
  * ユーザーの新規登録機能（管理者による手動登録）
  * ユーザーの編集機能（ユーザー名、メール、有効/無効、ロール変更）
  * ユーザーの削除機能

### 山データ（マスタ）管理

  * 山データの一覧表示（ページネーション対応）
  * 山データの新規登録機能（名前、英名、カナ名、標高、県名、緯度経度、距離、所要時間、累積標高、難易度、百名山フラグ、画像URL、出典等）
  * 山データの編集機能
  * 山データの削除機能

## 7. 非機能要件

### 技術スタック

  * **言語**: Java 17（Amazon Corretto）
  * **フレームワーク**: Spring Boot 3.x
  * **フロントエンド**: HTML5, CSS3, JavaScript (ES6+), Thymeleaf, Bootstrap 5
  * **地図ライブラリ**: Leaflet.js
  * **データベース**: MySQL（本番環境）/ H2（開発・テスト用）
  * **ORM**: Spring Data JPA
  * **認証・認可**: Spring Security
  * **決済**: Stripe Java SDK
  * **天気予報API**: OpenWeatherMap API
  * **ビルドツール**: Maven
  * **テスト**: JUnit 5, Mockito

### セキュリティ

  * Spring Securityによるロールベースアクセス制御（ROLE_GENERAL / ROLE_PREMIUM / ROLE_ADMIN）
  * パスワードのハッシュ化保存
  * メール認証によるアカウント有効化
  * 管理者画面への`@PreAuthorize`によるアクセス制限

### パフォーマンス・運用

  * ページネーションによる大量データの効率的な表示
  * 本番環境: Herokuへのデプロイ
  * 国際化（i18n）: 日本語・英語の対応

### 画像管理

  * 活動ログへの画像アップロード機能（BLOB形式でDB保存）
  * 山マスタへの画像URL設定（外部URL参照方式）
