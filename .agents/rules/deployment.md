【デプロイ前チェックリスト（補足項目）】

1. セキュリティ・秘匿情報の管理
   環境変数の外部化: application-prod.properties に直接 API キー（Stripe, OpenWeatherMap）を書き込んでいないか？
   OS側の環境変数、または AWS/Heroku 等のプラットフォーム側の「Config Vars」から読み込む設定になっていることを確認。
   エラーページの共通化: ホワイトラベル・エラーページ（デフォルトのスタックトレース）が露出しないよう、カスタムの 404/500 ページを用意しているか？
   SSL (HTTPS) 対応: 決済（Stripe）を扱うため、常時SSL化の設定が完了しているか。

2. データベース（DB）関連
   DBマイグレーション: H2（メモリ）から本番用（MySQL/PostgreSQL等）へ切り替える際のテーブル作成スクリプト（DDL）は準備できているか？
   ※Flyway や Liquibase 等を導入していない場合は、手動実行手順を確認。
   初期データ（マスターデータ）: 「日本百名山」の緯度・経度・標高データが本番DBにインポートされているか。

3. パフォーマンス・運用監視
   ログ出力設定: 開発用の DEBUG ログを INFO または WARN に引き上げているか（ログ肥大化防止）。
   静的リソースの最適化: CSS/JS のミニファイ化（圧縮）や、地図（Leaflet）のタイル読み込みが本番ドメインで制限されないかの確認。
   favicon/メタタグの設定: ブラウザのタブに表示されるアイコンや、共有時のOGP（タイトル・画像）が設定されているか。

4. 外部サービス連携（Stripe / 天気）
   Stripe 本番モードへの切り替え: テスト用キー（sk*test*...）から本番用キー（sk*live*...）への差し替え。
   Webhook エンドポイントの登録: Stripe からの通知先 URL を本番ドメイン（https://your-app.com/webhook）に更新しているか。
   OpenWeatherMap の制限確認: 本番のアクセス数で無料枠（1,000回/日）を超えないか、キャッシュ機能（@Cacheable）が有効に働いているか。

5. Java / Spring 特有の整理
   未使用コードの削除: コメントアウトされた古いロジックや、使用していないコントローラーの削除。
   冗長なコメントの削除
   依存関係（pom.xml）の精査: scope が test であるべきものが compile になっていないか（ビルドサイズの最適化）
